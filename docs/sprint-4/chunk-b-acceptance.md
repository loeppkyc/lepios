## Implementation notes (builder must read)

**Constraint B-1 — Finances: open group detection**
A financial event group is "open" (not yet transferred) when the `FundTransferStatus` field is absent from the response object. Do not check for a specific value — check for field presence: `!('FundTransferStatus' in group)` or `group.FundTransferStatus === undefined`. Closed groups have `FundTransferStatus: 'Transferred'` (or similar); open groups have the field omitted entirely.

**Constraint B-2 — Finances: CAD filter mandatory**
Filter `group.OriginalTotal?.CurrencyCode === 'CAD'` before summing. At least one open group has `CurrencyCode: MXN` with $0 amounts. Cross-currency sum produces a meaningless number.

**Constraint B-3 — FBA: `startDateTime` mandatory**
Without `startDateTime`, the endpoint returns the full lifetime FBA catalog (14,000+ records, 3+ minutes). Always pass `startDateTime` = 90 days ago. This filters to recently-updated SKUs only. The 90-day window covers all active listings and recent inbound shipments.

**Constraint B-4 — FBA: pagination params on every page**
On each paginated request (including those with `nextToken`), pass the full original param set: `details=true`, `granularityType=Marketplace`, `granularityId=A2EUQ1WTGCTBG2`, `marketplaceIds=A2EUQ1WTGCTBG2`, `startDateTime=<90-days-ago>`, `nextToken=<token>`. Passing only `nextToken` returns HTTP 400.

**Constraint B-5 — FBA: pagination token location**
`nextToken` is at `body.pagination.nextToken` (top-level response). It is NOT at `body.payload.pagination.nextToken`. Reading the wrong path produces a silent miss (no pagination, returns only first 50 records).

**Constraint B-6 — FBA: rate limiting**
Add 700ms delay between paginated requests. 429 observed at ~48 consecutive requests without throttle. Use `await sleep(700)` between pages. On 429, retry up to 3 times with 2-second backoff before failing.

**Constraint B-7 — FBA: fulfillable field path**
The unit count field is `inventoryDetails.fulfillableQuantity` (a direct number). Do not use `totalQuantity` — it includes all states. The panel shows fulfillable (sellable) units only. Inbound, reserved, and unsellable are excluded from the panel display.

**Constraint B-8 — FBA: 30-minute cache implementation**
Use `export const revalidate = 1800` on the `/api/business-review/fba-inventory` route handler. Do NOT use `force-dynamic` on this route. The response must include `fetchedAt: string` (ISO timestamp of when the data was actually fetched, not when the cached response was served). The panel component reads `fetchedAt` and displays "Last updated: X min ago".

**Constraint B-9 — Settlement route stays fast**
`/api/business-review/settlement` uses `export const dynamic = 'force-dynamic'`. No caching. The Finances endpoint returns one page of 82 groups — sub-second response. The panel fetches this route independently from the FBA route so the settlement number renders immediately without waiting for FBA inventory.
