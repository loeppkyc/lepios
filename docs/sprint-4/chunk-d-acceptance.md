## Timezone Rule

`server_modified` from Dropbox is UTC. **Convert to America/Edmonton before extracting
year and month.** Do not compare UTC timestamps directly against calendar month numbers.

**Example:** A file uploaded Apr 30 at 23:30 Mountain Time = May 1 at 05:30 UTC.

- UTC year+month: May 2026 → would land in May bucket (wrong)
- Edmonton year+month: April 2026 → lands in April bucket (correct)

**Tests must cover the end-of-month UTC/MT boundary.** Required test cases:

1. `server_modified = "2026-05-01T05:30:00Z"` → Edmonton = Apr 30 → **April 2026 bucket**
2. `server_modified = "2026-05-01T07:00:00Z"` → Edmonton = May 1 00:00 MT (MDT = UTC-6) → **May 2026 bucket**
3. A December 31 23:30 MT upload (= Jan 1 UTC) → must land in **December** not January

## Cell-State Rules

| Cell state | Definition                                                                                                               | What triggers it                                                                    |
| ---------- | ------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------- |
| **Green**  | ≥1 `.pdf` file in the account folder has `server_modified` (converted to America/Edmonton) with year == Y and month == M | Dropbox `files/list_folder` returns ≥1 PDF with matching Edmonton year+month        |
| **Red**    | 0 `.pdf` files match that Edmonton year+month                                                                            | Zero PDFs match — covers never uploaded, uploaded then deleted, mis-filed elsewhere |

**Presence rule is singular: one file is sufficient.** Multiple PDFs in the same month (e.g.,
a corrected statement re-upload) = still green. No requirement for exactly one file per month.

**No third state.** Every cell in both bands is green or red only. If an account was opened
mid-2025 and has no January statement, that cell is red — Colin knows his own account history.
No N/A state in v1.

**Filename is irrelevant.** Any `.pdf` with the correct Edmonton month counts as present.
No filename pattern check. No OCR. No content read.

**Why `server_modified` not `client_modified`:** `server_modified` is set by Dropbox on
upload and cannot be spoofed by the uploader; `client_modified` can be any date the
uploader sets. (Streamlit uses `server_modified` at `n8n_webhooks.py:65`.)

**Folders are flat (no YYYY-MM subdirectories).** Month assignment comes entirely from
`server_modified` converted to Edmonton time.

## Technical Approach

**New API route:** `app/api/business-review/statement-coverage/route.ts`

- Server-side only — credentials never exposed to client bundle
- `export const revalidate = 3600` (1 hour — Dropbox folders change slowly)
- Auth flow (port from Streamlit `dropbox_statements.py:77–80`):
  1. POST to `https://api.dropboxapi.com/oauth2/token` with `grant_type=refresh_token`
     using `DROPBOX_APP_KEY`, `DROPBOX_APP_SECRET`, `DROPBOX_REFRESH_TOKEN`
  2. Extract `access_token` from response
  3. Use access token for all `files/list_folder` calls
- 8 parallel `files/list_folder` calls (one per account)
- **No pagination** (`files/list_folder_continue` not needed): monthly bank statements ≈ 12
  files per account per year, never approaches Dropbox's 2000-entry cap
- Filter to `.pdf` entries; extract `server_modified` (UTC string); convert to
  America/Edmonton via `Intl.DateTimeFormat` or `date-fns-tz` (no new package preference —
  use whatever is already in the project; if nothing, use `Intl.DateTimeFormat`)
- Compute 2025 band (Jan–Dec 2025, fixed) and 2026 YTD band (Jan 2026 – current Edmonton month)
- For each account × month, check if any PDF's Edmonton year+month matches
- Response shape:
  ```ts
  {
    bands: [
      { label: "2025 · Tax Year", months: ["2025-01", ..., "2025-12"] },
      { label: "2026 · YTD",      months: ["2026-01", ..., "2026-04"] }  // dynamic
    ],
    accounts: Array<{
      key: string,
      label: string,
      coverage: Record<string, boolean>  // key = "YYYY-MM", value = present
    }>,
    fetchedAt: string  // ISO timestamp
  }
  ```

**npm package:** Add `dropbox` to `package.json` dependencies. Builder adds it.

**Timezone implementation note:** Do not use `.getMonth()` / `.getFullYear()` on a UTC Date
object — that returns UTC month, not Edmonton month. Use `Intl.DateTimeFormat` with
`timeZone: "America/Edmonton"` to extract local year and month from the UTC timestamp.

**New component:** `StatementCoverageGrid.tsx` in
`app/(cockpit)/business-review/_components/`

- Renders two-band grid with band labels
- Green cell: Design Council success color
- Red cell: Design Council error/danger color
- Loading and error states independent of other Business Review panels

**No n8n webhook.** Direct Dropbox API call per request, cached at `revalidate = 3600`.
