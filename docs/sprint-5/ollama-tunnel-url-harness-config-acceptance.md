## Background

`lib/ollama/client.ts` already has `hydrateOllamaConfig()` (reads OLLAMA_TUNNEL_URL from
harness_config, caches it for 5 min) and `getBaseUrl()` (returns harness_config value if
present, else process.env, else localhost). These are correct.

Three call sites still read `process.env.OLLAMA_TUNNEL_URL` directly when computing
`tunnel_used`, bypassing the harness_config cache. This means:

- If OLLAMA_TUNNEL_URL is in harness_config but NOT in process.env (the production runtime
  config pattern per S-L1), all three sites report `tunnel_used=false` even while routing
  calls through the tunnel.
- The daytime-tick never calls `hydrateOllamaConfig()` before its checks run, so
  `checkOllamaHealth()` → `healthCheck()` hits localhost when harness_config has the tunnel URL.

This chunk fixes all three sites.

## Gap 1 — `healthCheck()` tunnel_used fix

**File:** `lib/ollama/client.ts`

**Current code (inside `healthCheck()`):**

```typescript
const tunnelUsed =
  !!process.env.OLLAMA_TUNNEL_URL && process.env.OLLAMA_TUNNEL_URL !== 'http://localhost:11434'
```

**Replace with:**

```typescript
const baseUrl = getBaseUrl()
const tunnelUsed = !baseUrl.includes('localhost')
```

`getBaseUrl()` returns the harness_config cached value (if hydrated) or falls back to
process.env or localhost. Computing `tunnelUsed` from the resolved URL is the correct source
of truth — it matches exactly what `ollamaFetch()` uses as the target.

Note: `hydrateOllamaConfig()` must be called before `healthCheck()` for this to use the
harness_config value. The daytime-tick fix (Gap 2) ensures this. The smoke-test fix (Gap 3)
adds its own hydration call.

**Acceptance test addition (see Gap 4):** `tunnel_used=true` when harness_config URL is set
and process.env is absent.

### Step B — call hydrateOllamaConfig at start of runDaytimeTick

```typescript
// Hydrate harness_config cache so healthCheck() uses tunnel URL, not localhost

```

### Step C — fix getTunnelUsed()

**Current:**

```typescript
function getTunnelUsed(): boolean {
  const url = process.env.OLLAMA_TUNNEL_URL ?? ''
  return url.length > 0 && !url.includes('localhost')
}
```

**Replace with:**

```typescript
function getTunnelUsed(): boolean {
  return !getBaseUrl().includes('localhost')
}
```

After `hydrateOllamaConfig()` is called, `getBaseUrl()` returns the harness_config value.
Calling `getTunnelUsed()` AFTER `hydrateOllamaConfig()` is enforced by the ordering in Step B.
