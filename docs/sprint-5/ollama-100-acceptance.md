## Scope

Seven gap closures to move Ollama from 10% to 100% (reliability layer):

1. **Timeout: 60s → 15s** — generate calls fail 4× faster when Ollama is down
2. **Circuit breaker** (`lib/ollama/circuit.ts`, new) — after 3 consecutive failures in
   5 minutes, skip Ollama entirely; probe every 5 minutes for recovery
3. **Failure meta logging bug** — failure events currently log `meta: null`; fix to include
   `model`, `task`, and `error` on every failure
4. **Telegram alert on circuit transition** — one message when Ollama goes down; one on recovery
5. **F17: structured `routing_decision` field** on every `twin.ask` event
6. **F18: Ollama stats in morning_digest** — uptime %, fallback rate, latency p95
7. **`OLLAMA_TUNNEL_URL` startup warning** — one `agent_events` log if localhost fallback is
   active in production
8. **Model constants file** (`lib/ollama/models.ts`, new) — centralized model names as typed
   constants; clean integration point for GPU/fine-tuning work

**Daytime tick (Step 6.5) is explicitly out of scope.** It requires reliable Ollama, which
this doc builds. Daytime tick is a separate chunk after this ships and runs cleanly.

**One acceptance criterion:** After this ships, a Twin ask when Ollama is down completes in
under 20 seconds (not 62-70 seconds). The circuit opens after 3 consecutive failures. Colin
receives one Telegram message per outage, not one per failed call. `agent_events` failure
events include `meta.model` and `meta.error`.

## Gap 2 — Circuit Breaker

**File:** `lib/ollama/circuit.ts` (new)

### Circuit state logic

State is derived from `agent_events` — no new table, no migration.

```typescript
export type CircuitState = 'CLOSED' | 'OPEN' | 'HALF_OPEN'
export type CircuitOpenReason = 'server_unreachable' | 'model_not_loaded' | null

export interface CircuitStatus {
  state: CircuitState
  open_reason: CircuitOpenReason // null when CLOSED
  recent_failures: number // failures in last 5 minutes
  last_failure_at: string | null // ISO timestamp
  last_success_at: string | null
}

export async function getCircuitState(): Promise<CircuitStatus>
```

**State derivation query:**

```sql
SELECT
  COUNT(*) FILTER (
    WHERE status = 'failure'
    AND occurred_at > now() - interval '5 minutes'
  ) AS recent_failures,
  MAX(occurred_at) FILTER (WHERE status = 'failure') AS last_failure_at,
  MAX(occurred_at) FILTER (WHERE status = 'success') AS last_success_at
FROM agent_events
WHERE action = 'ollama.generate'
  AND occurred_at > now() - interval '30 minutes'
```

**State rules:**

- `recent_failures >= 3` AND `last_failure_at > now() - 5 min` → **OPEN**
- `recent_failures >= 3` AND `last_failure_at <= now() - 5 min` → **HALF_OPEN** (try probe)
- `recent_failures < 3` → **CLOSED**

### Integration into `generate()`

**File:** `lib/ollama/client.ts` — beef up `generate()`:

```typescript
export async function generate(
  prompt: string,
  opts: GenerateOptions = {}
): Promise<GenerateResult> {
  // 1. Check circuit state (adds ~15ms but saves 15s+ on OPEN circuit)
  const circuit = await getCircuitState()

  if (circuit.state === 'OPEN') {
    void logEvent('ollama', 'ollama.circuit_skip', {
      actor: 'system',
      status: 'skipped',
      meta: { reason: 'circuit_open', recent_failures: circuit.recent_failures },
    })
    throw new OllamaUnreachableError('circuit open — skipping Ollama')
  }

  if (circuit.state === 'HALF_OPEN') {
    // Probe: GET /api/tags, then verify model is in the response
    const health = await healthCheck() // returns { reachable, models: string[], ... }
    const targetModel = opts.model ?? autoSelectModel(opts.task ?? 'general')
    const modelLoaded = health.reachable && health.models.includes(targetModel)

    if (!health.reachable) {
      void logEvent('ollama', 'ollama.circuit_probe_failed', {
        actor: 'system',
        status: 'failure',
        meta: { state: 'HALF_OPEN', reason: 'server_unreachable', model: targetModel },
      })
      throw new OllamaUnreachableError('half-open probe failed: server unreachable')
    }
    if (!modelLoaded) {
      void logEvent('ollama', 'ollama.circuit_probe_failed', {
        actor: 'system',
        status: 'failure',
        meta: {
          state: 'HALF_OPEN',
          reason: 'model_not_loaded',
          model: targetModel,
          available_models: health.models,
        },
      })
      throw new OllamaUnreachableError(`half-open probe failed: model ${targetModel} not loaded`)
    }
    // Probe succeeded — server reachable + model present; fall through to generate
  }

  // 2. Proceed with generate (existing logic, timeout now 15s)
  // ... existing generate code ...
}
```

**Circuit state transition logging** (fire-and-forget, in `getCircuitState()`):

When state transitions from CLOSED to OPEN (first time `recent_failures >= 3`):

```typescript
void logEvent('ollama', 'ollama.circuit_open', {
  actor: 'system',
  status: 'warning',
  meta: {
    recent_failures: circuit.recent_failures,
    last_failure_at: circuit.last_failure_at,
    open_reason: circuit.open_reason, // 'server_unreachable' | 'model_not_loaded'

})
```

When state transitions from OPEN/HALF_OPEN to CLOSED (probe success):

```typescript
void logEvent('ollama', 'ollama.circuit_closed', {
  actor: 'system',
  status: 'success',
  meta: { was_open_since: circuit.last_failure_at },
})
```

**Transition detection:** Keep a module-level variable `let _lastState: CircuitState = 'CLOSED'`.
Compare with new state on each call. If changed, log the transition event. (Module-level
is per process instance — good enough for Vercel; the agent_events logs serve as the
durable record.)

## Kill Signals

1. **Circuit check adds >50ms to happy-path generate calls** — the Supabase SELECT should
   complete in <15ms with the existing index on `(action, occurred_at)`. If slower, add a
   covering index or cache the result for 30 seconds in a module-level variable. Stop and
   flag before shipping if p99 > 50ms in local testing.

2. **Circuit state derived from stale events** — the query uses `occurred_at > now() - 30 minutes`.
   If Supabase clock skew or timezone difference causes unexpected results in the 30-minute
   window, the circuit will misbehave. Builder must verify the query returns expected counts
   in a unit test before wiring it into `generate()`.

3. **`outbound_notifications` insert blocks generate()** — the Telegram alert insert must be
   fire-and-forget (`void insertNotification(...)`). Never await it inline in the generate path.

4. **Morning digest breaks if Ollama query fails** — wrap the Ollama stats query in try/catch.
   On failure, the digest line reads "Ollama: stats unavailable" — do NOT let a failed stats
   query kill the entire digest.
